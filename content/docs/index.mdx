## Code Blocks

Code blocks support syntax highlighting for multiple languages. Use tabs to switch between different implementations:

<CodeBlockTabs>
  <CodeBlockTabsList>

    <CodeBlockTabsTrigger value="python">Python</CodeBlockTabsTrigger>

  </CodeBlockTabsList>
  <CodeBlockTab value="javascript">
    ```javascript
    function helloWorld() {
      console.log("Hello, World!");
    }
    
    helloWorld();
    ```
  </CodeBlockTab>
  
  <CodeBlockTab value="python">
    ```python
    def hello_world():
        print("Hello, World!")
    
    hello_world()
    ```
  </CodeBlockTab>
  
  <CodeBlockTab value="typescript">
    ```typescript
    function helloWorld(): void {
      console.log("Hello, World!");
    }
    
    helloWorld();
    ```
  </CodeBlockTab>
</CodeBlockTabs>

## Callouts

Callouts highlight important information with different semantic types:

### Basic Callout Component

<Callout type="info" title="Information">
  Use info callouts to provide additional context or helpful tips to your readers.
</Callout>

<Callout type="warning" title="Important Warning">
  Warning callouts draw attention to potential issues or breaking changes users should be aware of.
</Callout>

<Callout type="success" title="Success Message">
  Success callouts celebrate completed actions or positive outcomes in your documentation.
</Callout>

<Callout type="error" title="Error Alert">
  Error callouts highlight critical issues, failures, or dangerous operations that require caution.
</Callout>

### Callout Container (Advanced)

For more control over the structure, use CalloutContainer with separate title and description:

<CalloutContainer type="info">
  <CalloutTitle>Getting Started</CalloutTitle>
  <CalloutDescription>
    This approach allows you to include multiple paragraphs and complex content within a single callout. You can even add links like [this one](/) for better navigation.
  </CalloutDescription>
</CalloutContainer>

<CalloutContainer type="warning">
  <CalloutTitle>Breaking Changes</CalloutTitle>
  <CalloutDescription>
    Version 2.0 introduces several breaking changes. Make sure to review the migration guide before upgrading your project.
  </CalloutDescription>
</CalloutContainer>

### Best Practices

Follow these best practices when using Fumadocs:

1. **Code Blocks** - Use tabs to show multiple languages or implementations
2. **Callouts** - Choose the right semantic type (info, warning, success, error)
3. **Cards** - Group related navigation items together
4. **Tables** - Keep content concise and scannable
5. **Headings** - Use proper hierarchy (h1, h2, h3, etc.)
6. **Links** - Use descriptive text for accessibility
